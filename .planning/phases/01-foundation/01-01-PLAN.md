<objective>
Scaffold the FastAPI backend with the domain-driven structure (auth/listings/users/upload), define every database table required by the marketplace in Alembic migration 001 (PostGIS + JSONB attributes + spatial index + seeded vehicle_makes), and configure Railway to deploy it with migrations on every release.

Purpose: Phase 1 establishes the database schema permanently — every later phase reads/writes against these tables. The schema decisions made here cannot be cheaply reverted, so this plan is exhaustive and creates all four tables in a single migration. The Railway PostGIS quirk (standard Postgres lacks superuser to install PostGIS) is the single biggest deployment risk and is mitigated by provisioning the PostGIS template up front.

Output: A `/backend/` directory that boots locally with `uvicorn app.main:app --reload`, a migration 001 that applies to any PostGIS database, and a `railway.toml` that runs `alembic upgrade head` before each deploy.
</objective>

from sqlalchemy import engine_from_config, pool

    fileConfig(config.config_file_name)

def run_migrations_offline() -> None:
    url = config.get_main_option("sqlalchemy.url")
    context.configure(
        url=url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
        include_object=alembic_helpers.include_object,
        render_item=alembic_helpers.render_item,
    )
    with context.begin_transaction():
        context.run_migrations()


def run_migrations_online() -> None:
    connectable = engine_from_config(
        config.get_section(config.config_ini_section, {}),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )

    with connectable.connect() as connection:
        context.configure(
            connection=connection,
            target_metadata=target_metadata,
            include_object=alembic_helpers.include_object,
            render_item=alembic_helpers.render_item,
            include_schemas=True,
        )
        with context.begin_transaction():
            context.run_migrations()


if context.is_offline_mode():
    run_migrations_offline()
else:
    run_migrations_online()
